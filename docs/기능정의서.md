# 기능정의서 — 대진밥핏

> 기능별 입력값, 동작, 예외 상황 체크리스트입니다.

## 기능 목록

| ID | 기능 | 우선순위 | 상태 |
| --- | --- | --- | --- |
| F1 | 학식 메뉴 검색 | 높음 | 준비 |
| F2 | 영양·열량 분석 | 높음 | 준비 |
| F3 | 회원가입·로그인 | 높음 | 준비 |
| F4 | AI 영양 조언 | 높음 | 준비 |
| F5 | 식사 기록 | 중간 | 준비 |

## F1. 학식 메뉴 검색

- 목적: 대진대 학식 메뉴를 빠르게 찾는다.
- 사용자: 대진대 재학생
- 진입 화면: 홈

### 입력값

| 입력값 | 필수? | 규칙 |
| --- | --- | --- |
| 검색어 | 아니오 | 1자 이상, 메뉴명 부분 일치 |
| 날짜 | 아니오 | 기본값: 오늘 |

### 동작

1. 홈 진입 시 대진대학교 학식 페이지를 웹 크롤링해 오늘 학식 목록을 불러온다.
2. 사용자가 검색어를 입력하면 메뉴명·식당명 기준으로 필터링한다.
3. 카드 목록(메뉴명, 가격, 식당, 식사 구분)을 표시한다.
4. 카드 탭 시 메뉴 상세(F2)로 이동한다.

### 예외 상황

- API 실패: 대진대 샘플 식단 데이터로 폴백 + "실시간 연동 준비 중" 안내
- 결과 없음: "검색 결과가 없어요. 다른 키워드를 입력해 보세요."
- 오늘 식단 미등록: "오늘 등록된 메뉴가 없어요."

## F2. 영양·열량 분석

- 목적: 선택한 메뉴의 영양 성분을 표시한다.
- 진입: 메뉴 상세 화면

### 입력값

| 입력값 | 필수? | 규칙 |
| --- | --- | --- |
| 메뉴명 | 예 | 영양 분석 키워드로 사용 |

### 동작

1. 메뉴명에서 핵심 키워드(예: "돈까스", "제육")를 추출한다.
2. 식품의약품안전처 API를 호출하지 않고 메뉴명 기반 규칙으로 영양 정보를 분석한다.
3. 열량, 단백질, 탄수화물, 지방, 나트rium 등을 카드·바 형태로 표시한다.
4. F4 AI 조언을 함께 표시한다.

### 예외 상황

- API 키 없음: 추정 영양값(메뉴 유형별 기본값) + "참고용 추정치입니다"
- 매칭 실패: 유사 메뉴 또는 카테고리 평균값 사용
- API 한도 초과: 캐시된 결과 또는 추정값

## F3. 회원가입·로그인

- 목적: 개인 식사 기록 및 프로필 관리
- 진입: 하단 탭 프로필, 또는 기록하기 시

### 입력값

| 입력값 | 필수? | 규칙 |
| --- | --- | --- |
| 이메일 | 예 | 이메일 형식 |
| 비밀번호 | 예 | 6자 이상 |
| 이름 | 가입 시 예 | 1자 이상 |
| 학과 | 아니오 | 자유 입력 |

### 동작

1. Supabase Auth로 이메일 가입/로그인 처리.
2. 가입 시 profiles 테이블에 이름·학과 저장.
3. 로그인 성공 시 이전 화면 또는 홈으로 이동.

### 예외 상황

- 이메일 중복: "이미 가입된 이메일입니다."
- 비밀번호 불일치: "비밀번호를 확인해 주세요."
- Supabase 미설정: "서비스 설정 중입니다. .env.local을 확인해 주세요."

## F4. AI 영양 조언

- 목적: 영양 데이터 기반 실용 조언 제공
- 진입: 메뉴 상세, AI 조언 탭

### 입력값

| 입력값 | 필수? | 규칙 |
| --- | --- | --- |
| 메뉴 영양 정보 | 예 | F2 결과 |
| (로그인 시) 오늘 누적 기록 | 아니오 | meal_logs 합산 |

### 동작

1. 단백질·칼로리·나트rium 기준으로 규칙 기반 조언 생성.
2. 예: 단백질 < 15g → "단백질이 부족해요. 닭가슴살·두부 메뉴를 추천해요."
3. (선택) OPENAI_API_KEY 있으면 LLM으로 자연어 조언 보강.
4. 조언 카드 2~4개를 한국어로 표시.

### 예외 상황

- 영양 정보 없음: 일반적인 식단 균형 조언만 표시
- OpenAI 실패: 규칙 기반 조언만 표시 (서비스 중단 없음)

## F5. 식사 기록

- 목적: 먹은 학식을 기록해 AI 개인화에 활용
- 진입: 메뉴 상세 "기록하기" 버튼

### 입력값

| 입력값 | 필수? | 규칙 |
| --- | --- | --- |
| 메뉴명 | 예 | 자동 입력 |
| 영양 수치 | 예 | F2 결과 자동 |

### 동작

1. 로그인 확인 → 미로그인 시 로그인 화면 이동.
2. meal_logs에 user_id, 메뉴, 영양, 시간 저장.
3. "기록했어요!" 토스트 후 내 기록 탭에서 확인.

### 예외 상황

- 미로그인: 로그인 유도
- 저장 실패: "저장에 실패했어요. 다시 시도해 주세요."

## 공통 규칙

- 사용자 문구는 한국어.
- API 키는 `.env.local`에만, 코드/깃에 넣지 않음.
- 본인 meal_logs만 조회·삭제 (Supabase RLS).
- 모바일 최대 폭 480px.

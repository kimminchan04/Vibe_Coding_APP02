# 유저플로우 — 대진밥핏

> 대진대학교 학생이 앱을 사용하는 화면 흐름입니다.

## 1. 주요 사용자 흐름

```text
첫 방문
  -> 홈(검색)
  -> 메뉴 목록 / 검색 결과
  -> 메뉴 상세 (영양 + AI 조언)
  -> (선택) 식사 기록
  -> 완료
```

## 2. 화면 목록

| 화면 | 하는 일 | 로그인 필요? | 다음 이동 |
| --- | --- | --- | --- |
| 홈(검색) | 오늘 학식 + 메뉴 검색 | 아니오 | 검색 결과 / 메뉴 상세 |
| 검색 결과 | 키워드 매칭 메뉴 목록 | 아니오 | 메뉴 상세 |
| 메뉴 상세 | 영양·열량 + AI 조언 | 아니오 | 식사 기록(로그인 시) |
| AI 조언 | 오늘/주간 영양 요약 조언 | 아니오(기본) / 예(개인화) | 메뉴 상세 |
| 내 기록 | 식사 기록 목록 | 예 | 메뉴 상세 / 삭제 |
| 로그인 | 이메일 로그인 | - | 이전 화면 |
| 회원가입 | 이메일 가입 + 이름·학과 | - | 홈 |
| 프로필 | 이름·학과·목표 칼로리 설정 | 예 | 로그아웃 |

## 3. 핵심 시나리오

### 시나리오 A. 점심 메뉴 검색 후 영양 확인

1. 홈에 들어온다.
2. 검색창에 "돈까스"를 입력한다.
3. 검색 결과에서 `[돈카츠락] 왕등심돈까스`를 탭한다.
4. 영양 정보(칼로리, 단백질 등)와 AI 조언("단백질이 풍부해요")을 확인한다.

### 시나리오 B. 회원이 식사 기록 후 AI 조언 확인

1. 로그인한다.
2. 점심 메뉴 상세에서 "식사 기록하기"를 누른다.
3. AI 조언 탭에서 오늘 누적 영양과 "저녁은 가벼운 메뉴를 추천해요" 조언을 본다.

### 시나리오 C. 비회원이 기본 기능 사용

1. 로그인 없이 홈에서 오늘 학식 목록을 본다.
2. 메뉴 상세에서 영양 정보와 기본 AI 조언을 확인한다.
3. 기록하기를 누르면 로그인 화면으로 안내된다.

## 4. 예외 흐름

- API 키 없음 / API 오류: 샘플 데이터 + "영양 정보는 참고용입니다" 안내
- 검색 결과 없음: "검색 결과가 없어요" + 다른 키워드 제안
- 로그인 필요: "로그인하면 식사 기록을 남길 수 있어요" + 로그인 버튼
- 저장 실패: "잠시 후 다시 시도해 주세요"
